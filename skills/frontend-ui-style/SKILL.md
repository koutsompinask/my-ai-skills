# Frontend UI Style

Push frontend work toward a clear visual point of view. Avoid safe default layouts, generic SaaS styling, and interchangeable component assemblies.

## Workflow

1. Read [`references/style-guidelines.md`](references/style-guidelines.md) before making substantial UI changes.
2. Determine whether the project already has a design system or strong product language.
3. Preserve and extend established patterns when they exist.
4. If the product language is weak or undefined, introduce a deliberate visual direction instead of falling back to generic defaults.
5. Apply the review checklist before finishing any meaningful frontend task.

## Design Rules

- Give the interface a recognizable point of view; choose a mood and make typography, spacing, surfaces, and motion reinforce it.
- Avoid default-feeling stacks, flat backgrounds, weak contrast, and boilerplate card grids unless the product already relies on them.
- Use typography as a design tool, not just a rendering default.
- Build atmosphere with layered backgrounds, color systems, and shape, not only with white space.
- Use motion to clarify hierarchy, transitions, or state changes; avoid decorative animation with no UX purpose.
- Keep accessibility intact while increasing visual character.

## Existing Systems

- When a project already has a coherent design language, do not replace it with unrelated stylistic ideas.
- Improve weak areas by working inside the current system: strengthen spacing, contrast, hierarchy, states, and composition before inventing new primitives.
- Introduce new reusable primitives only when a pattern repeats or the existing system is clearly insufficient.

## Review Checklist

- Check whether the screen has a clear visual identity rather than looking like a default starter template.
- Check typography choice, scale, and rhythm.
- Check whether the color palette and surfaces create depth and atmosphere.
- Check whether spacing and alignment feel intentional.
- Check whether motion supports comprehension and respects reduced-motion expectations.
- Check responsiveness on mobile and desktop.
- Check accessibility basics: contrast, focus visibility, readable states, and keyboard reachability.
